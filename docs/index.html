Proyecto

Caminar el método

5 km/h parte de una velocidad específica: el paso peatonal. Desde ahí, la ciudad aparece como una serie de permisos, interrupciones y ajustes que rara vez entran en los mapas de movilidad.

El proyecto construye un sistema para registrar esas modificaciones y probar cómo cambian cuando se traducen a matriz, textil, interfaz y sonido.

Pregunta

Quién modifica el paso

La investigación pregunta qué fuerzas toman mando sobre el movimiento peatonal y cómo cambia esa relación según el territorio.

Sistema

Caminar, registrar, traducir

El sistema convierte cada modificación del paso en una unidad legible: acción, agente, ruta, intensidad y posición dentro del recorrido.

Salida

Un proyecto transmedia

Las salidas no ilustran el tema: cada soporte prueba una forma distinta de leer, comparar o activar la misma información.

Método

Observar desde la velocidad del cuerpo

Protocolo

Cada ruta se recorre sin buscar eficiencia ni velocidad máxima. La atención se pone en los momentos donde el trayecto deja de ser continuo: una pausa, una reducción de velocidad, una aceleración o un cambio de dirección.

Qué se registra

Cada anotación conserva cinco datos: ubicación dentro de la ruta, acción corporal, agente que la provoca, intensidad percibida y observación breve de contexto.

Sistema

Sistema de lectura

El sistema separa dos capas que en campo aparecen mezcladas: la acción que modifica el paso y el agente que produce esa modificación. Esa separación permite comparar rutas sin borrar sus condiciones particulares.

Eventos

  • Esperarel cuerpo queda detenido por una condición externa y debe leer cuándo volver a avanzar.
  • Frenarel paso pierde intensidad para negociar un borde, una cercanía o una posible fricción.
  • Acelerarla velocidad aumenta para cruzar, sincronizarse o salir de una presión del entorno.
  • Desviarsela ruta cambia para rodear una interrupción sin abandonar el recorrido.

Agentes

  • Infraestructurabanquetas, cruces, guías, desniveles y límites.
  • Vehículosautos, motos, transporte y velocidad motorizada.
  • Humanosmultitud, encuentros, cuerpos cercanos y flujos peatonales.
  • No humanosvegetación, animales, clima, objetos y materialidad viva.
  • Decisión propiaelección del cuerpo para pausar, rodear, mirar o seguir.

Traducciones

Transformar el registro en medios

Las traducciones usan la misma base de datos: ruta, evento, agente e intensidad. La matriz es el espiral: ordena las acciones de quietud a desvío y los agentes de lo impersonal a lo personal. Después esa estructura se prueba como textil, interfaz y sonido.

01

Experiencia a datos

La experiencia se vuelve registro cuando se decide una unidad mínima: un momento donde el paso cambia y puede localizarse dentro de la ruta.

02

Datos a eventos

Los registros se normalizan con el mismo vocabulario para que rutas distintas puedan compararse sin depender solo de una crónica personal.

03

Eventos a puntadas

Cada evento se asigna a un signo y a una posición. La repetición de signos produce una superficie que permite ver acumulaciones, vacíos y cambios de ritmo.

04

Puntadas a matriz

La matriz es el espiral: las acciones avanzan de quietud a desvío y los agentes de lo más impersonal a lo más propio.

05

Matriz a sonido

La matriz funciona como partitura: permite leer y activar los datos otra vez desde el cuerpo, reconociendo las negociaciones del recorrido a través de sonidos dominantes.

Evolución

Cómo evolucionó el proyecto

01

Partir del campo

Empezamos caminando, anotando incomodidades, miedo, pausas y obstáculos que al inicio parecían detalles sueltos del recorrido.

02

Separar el sistema

Tuvimos que separar qué le pasaba al cuerpo de quién lo provocaba. Ahí aparecieron las acciones y los agentes como dos capas distintas del mismo registro.

03

Encontrar la matriz

La matriz empezó como forma de ordenar, pero terminó entendida como espiral: de quietud a desvío, y de fuerzas impersonales a decisiones más propias.

04

Probar traducciones

Probamos el sistema en textil, interfaz y sonido. Cada traducción mostró algo distinto: acumulación, comparación, dominancias, pausas o formas de volver a leer los datos.

05

Regresar al cuerpo

El método se tuvo que ajustar muchas veces porque caminar era nuestra forma de investigar y también lo que estábamos estudiando. El sonido nos permitió cerrar volviendo al cuerpo.

Activación

Del sistema a la interfaz

La interfaz permite comprobar relaciones específicas: qué agente domina una ruta, qué acción se repite más y qué cambia al comparar territorios. Su función es hacer operable el método, no solo mostrarlo.

Archivo

Archivo vivo

El archivo no reúne únicamente evidencias del recorrido, sino rastros del conocimiento producido al caminar: encuentros, anomalías, decisiones, ajustes metodológicos y fragmentos que el sistema analítico no captura por completo.
